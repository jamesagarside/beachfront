# Domain docs

**Single-context** repo.

- Domain glossary: `CONTEXT.md` at the repo root.
- Architectural decisions: `docs/adr/`.

Skills that read domain language (`to-issues`, `triage`, `diagnose`,
`improve-codebase-architecture`, `tdd`) should read `CONTEXT.md` for vocabulary and
respect the ADRs in `docs/adr/` for the area they touch.
